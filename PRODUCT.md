# Relay AI Product Context

## Register

Product.

## Users

Relay AI serves people who use AI coding agents and desktop assistants and want to connect them to multiple model providers. The audience includes both experienced developers and people who prefer a visual, guided workflow over manual configuration.

## Product Purpose

Relay AI gives users one place to configure providers, choose models, launch supported AI tools, manage favorites, and run local API gateways. It reduces provider-specific setup while keeping credentials and local configuration under the user's control.

## Brand Personality

Clear, capable, direct, and technically confident. The interface should feel polished and calm without hiding important provider, authentication, or connection details.

## Anti-References

- Generic SaaS decoration that competes with the task.
- Inconsistent provider identity, such as showing a brand logo in one view and a generic initial in another.
- Ambiguous connection states or actions with hidden consequences.
- Dense configuration screens that assume the user already knows provider-specific terminology.
- Motion or visual effects that slow down routine work.

## Design Principles

1. Keep identity consistent. A provider, model, application, or status should look and read the same everywhere it appears.
2. Make the next action obvious. Primary workflows should be understandable without documentation.
3. Reveal complexity progressively. Show essential choices first and advanced configuration only when it is relevant.
4. Prefer explicit state. Connection, loading, success, warning, and error states should be visible and actionable.
5. Preserve user control. Explain authentication and local gateway behavior clearly, especially when credentials or external services are involved.
6. Use a focused technical visual system. Deep navy surfaces, electric indigo accents, compact spacing, and restrained motion support the work instead of decorating it.

## Accessibility and Inclusion

- Support keyboard navigation and visible focus states for every interactive control.
- Maintain readable contrast in dark and light themes.
- Respect reduced-motion preferences.
- Use plain language for provider setup and authentication flows.
- Keep layouts usable at narrow viewport sizes and avoid relying on color alone to communicate state.
